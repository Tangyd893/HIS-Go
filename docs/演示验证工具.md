# HIS-Go 演示验证工具

## 概述

本文档说明如何使用 HIS-Go 演示验证工具来验证演示环境是否正常工作。

## 工具说明

演示验证工具是一个 Bash 脚本，用于自动化验证演示环境的各个方面，包括：

1. 服务健康检查
2. 前端访问验证
3. API 接口测试
4. 功能流程验证

## 使用方法

### 1. 验证管理端演示

```bash
# 使用 Makefile
make verify-admin

# 或直接运行脚本
./scripts/demo-verify.sh admin
```

### 2. 验证患者端演示

```bash
# 使用 Makefile
make verify-patient

# 或直接运行脚本
./scripts/demo-verify.sh patient
```

### 3. 测试 API 接口

```bash
# 使用 Makefile
make demo-verify

# 或直接运行脚本
./scripts/demo-verify.sh api
```

### 4. 验证所有演示环境

```bash
# 使用 Makefile
make demo-verify

# 或直接运行脚本
./scripts/demo-verify.sh all
```

## 验证内容

### 管理端演示验证

1. **服务健康检查**
   - Gateway (端口 8080)
   - Auth (端口 8081)
   - User (端口 8082)
   - Registration (端口 8083)
   - Clinic (端口 8084)
   - Prescription (端口 8085)
   - Billing (端口 8086)
   - Pharmacy (端口 8087)
   - Schedule (端口 8090)
   - System (端口 8096)

2. **前端访问验证**
   - Nginx 代理正常
   - 管理端前端可访问 (http://localhost/admin)

### 患者端演示验证

1. **服务健康检查**
   - Gateway (端口 8080)
   - Auth (端口 8081)
   - User (端口 8082)
   - Registration (端口 8083)
   - Schedule (端口 8090)
   - Prescription (端口 8085)
   - Examination (端口 8088)
   - Followup (端口 8092)
   - Health Record (端口 8093)

2. **前端访问验证**
   - Nginx 代理正常
   - 患者端前端可访问 (http://localhost/patient)

### API 接口测试

1. **登录接口测试**
   - 测试 POST /api/auth/login
   - 验证登录成功
   - 获取 Token

2. **用户信息接口测试**
   - 测试 GET /api/auth/current
   - 验证获取用户信息成功

## 故障排查

### 1. 服务健康检查失败

```bash
# 检查服务状态
docker ps

# 查看服务日志
./scripts/demo-admin.sh logs his-auth

# 检查端口占用
netstat -tulpn | grep :8081
```

### 2. 前端访问失败

```bash
# 检查 Nginx 状态
docker compose -f docker/docker-compose.demo-admin.yml ps nginx

# 查看 Nginx 日志
docker compose -f docker/docker-compose.demo-admin.yml logs nginx

# 检查前端构建产物
ls -la frontend/his-web-admin-react/dist/
```

### 3. API 接口测试失败

```bash
# 检查 Gateway 状态
curl http://localhost:8080/health

# 测试登录接口
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"demo-admin","password":"demo123"}'

# 查看 Gateway 日志
./scripts/demo-admin.sh logs his-gateway
```

## 自定义验证

### 1. 修改验证脚本

如果需要添加或修改验证内容，可以编辑以下文件：

- `scripts/demo-verify.sh` - 演示验证工具
- `scripts/verify-demo.sh` - 演示环境验证脚本

### 2. 添加新的验证项

在 `demo-verify.sh` 中添加新的验证函数：

```bash
# 验证新功能
verify_new_feature() {
    print_info "验证新功能..."
    
    # 添加验证逻辑
    
    if [ 验证通过 ]; then
        print_success "新功能验证通过"
        return 0
    else
        print_error "新功能验证失败"
        return 1
    fi
}
```

然后在主函数中调用：

```bash
main() {
    case "${1:-help}" in
        # ... 其他命令 ...
        new-feature)
            verify_new_feature
            ;;
        # ... 其他命令 ...
    esac
}
```
