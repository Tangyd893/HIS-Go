# HIS-Go 演示环境验证指南

## 概述

本文档说明如何验证 HIS-Go 演示环境是否正常工作。

## 前置条件

- Docker 20.10+
- Docker Compose v2.0+
- curl 命令行工具

## 快速验证

### 1. 启动演示服务

```bash
# 启动管理端演示
make demo-admin

# 或者启动患者端演示
make demo-patient
```

### 2. 验证演示环境

#### 验证管理端演示

```bash
make verify-admin
```

或者直接运行脚本：

```bash
./scripts/verify-demo.sh admin
```

#### 验证患者端演示

```bash
make verify-patient
```

或者直接运行脚本：

```bash
./scripts/verify-demo.sh patient
```

## 验证内容

### 管理端演示验证

验证脚本会检查以下服务：

1. **基础设施服务**
   - PostgreSQL (端口 5432)
   - Redis (端口 6379)
   - RabbitMQ (端口 5672)

2. **微服务**
   - Gateway (端口 8080)
   - Auth (端口 8081)
   - User (端口 8082)
   - Registration (端口 8083)
   - Clinic (端口 8084)
   - Prescription (端口 8085)
   - Billing (端口 8086)
   - Pharmacy (端口 8087)
   - Schedule (端口 8090)
   - System (端口 8096)

3. **前端**
   - Nginx 代理
   - 管理端前端 (http://localhost/admin)

### 患者端演示验证

验证脚本会检查以下服务：

1. **基础设施服务**
   - PostgreSQL (端口 5432)
   - Redis (端口 6379)

2. **微服务**
   - Gateway (端口 8080)
   - Auth (端口 8081)
   - User (端口 8082)
   - Registration (端口 8083)
   - Schedule (端口 8090)
   - Prescription (端口 8085)
   - Examination (端口 8088)
   - Followup (端口 8092)
   - Health Record (端口 8093)

3. **前端**
   - Nginx 代理
   - 患者端前端 (http://localhost/patient)

## 手动验证

### 1. 检查服务状态

```bash
# 查看 Docker 容器状态
docker ps

# 查看演示服务状态
make demo-status
```

### 2. 检查服务健康

```bash
# 检查 Gateway 健康状态
curl http://localhost:8080/health

# 检查各服务健康状态
curl http://localhost:8081/health  # Auth
curl http://localhost:8082/health  # User
```

### 3. 检查前端访问

```bash
# 检查管理端前端
curl -I http://localhost/admin

# 检查患者端前端
curl -I http://localhost/patient
```

### 4. 测试 API 接口

```bash
# 登录获取 Token
TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"demo-admin","password":"demo123"}' | jq -r '.data.token')

# 使用 Token 访问 API
curl -H "Authorization: Bearer $TOKEN" http://localhost:8080/api/auth/current
```

## 故障排查

### 1. 服务启动失败

```bash
# 查看服务日志
make demo-admin-logs
# 或
make demo-patient-logs

# 查看特定服务日志
./scripts/demo-admin.sh logs his-auth
```

### 2. 端口冲突

```bash
# 检查端口占用
netstat -tulpn | grep :8080
netstat -tulpn | grep :5432

# 停止占用端口的进程
sudo kill -9 <PID>
```

### 3. 数据库连接失败

```bash
# 检查 PostgreSQL 状态
docker compose -f docker/docker-compose.demo-admin.yml ps postgresql

# 查看 PostgreSQL 日志
docker compose -f docker/docker-compose.demo-admin.yml logs postgresql
```

### 4. 前端无法访问

```bash
# 检查 Nginx 配置
docker compose -f docker/docker-compose.demo-admin.yml exec nginx nginx -t

# 查看 Nginx 日志
docker compose -f docker/docker-compose.demo-admin.yml logs nginx
```

## 性能测试

### 1. 并发测试

```bash
# 安装 Apache Bench
sudo apt install apache2-utils

# 测试 Gateway 性能
ab -n 100 -c 10 http://localhost:8080/health
```

### 2. 资源监控

```bash
# 查看容器资源使用
docker stats

# 查看系统资源
htop
```

## 清理环境

### 1. 停止演示服务

```bash
# 停止管理端演示
make demo-admin-stop

# 停止患者端演示
make demo-patient-stop
```

### 2. 清理数据

```bash
# 停止服务并删除数据卷
docker compose -f docker/docker-compose.demo-admin.yml down -v

# 清理构建产物
make clean
```
