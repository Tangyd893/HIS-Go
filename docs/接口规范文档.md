# HIS-Go API 接口规范文档

## 一、约定

### 1.1 统一响应格式

```json
{
  "code": 0,
  "message": "成功",
  "data": {}
}
```

- `code=0` 表示成功，非0表示业务错误
- `data` 为业务数据，失败时可省略

### 1.2 分页响应

```json
{
  "code": 0,
  "message": "成功",
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

### 1.3 错误码体系

| 范围 | 类别 | 示例 |
|------|------|------|
| 0 | 成功 | 0 |
| 10000-10099 | 通用系统错误 | 10001-参数无效 10002-未认证 10003-无权限 |
| 20000-29999 | 业务错误 | 20001-用户不存在 20003-号源已满 20006-库存不足 |
| 30000-39999 | 第三方错误 | 30001-支付失败 30002-医保接口异常 |
| 90000-99999 | 未知错误 | 90000-未知系统错误 |

### 1.4 HTTP 状态码映射

| 错误码 | HTTP 状态码 |
|--------|-----------|
| 0 | 200 |
| 10001 | 400 Bad Request |
| 10002 | 401 Unauthorized |
| 10003 | 403 Forbidden |
| 10004 | 404 Not Found |
| 10005 | 409 Conflict |
| 10006 | 408 Request Timeout |
| 10009 | 429 Too Many Requests |
| 10007 | 500 Internal Server Error |
| 10008 | 503 Service Unavailable |

## 二、认证接口

### 2.1 登录

```
POST /api/auth/login
Content-Type: application/json

Request:
{
  "username": "demo-admin",
  "password": "demo123"
}

Response:
{
  "code": 0,
  "message": "成功",
  "data": {
    "token": "eyJhbGciOiJSUzI1NiIs...",
    "user": {
      "id": "uuid",
      "username": "demo-admin",
      "realName": "系统管理员",
      "role": "admin",
      "deptId": "dept-001",
      "perms": ["user:list", "system:admin"]
    }
  }
}
```

### 2.2 刷新令牌

```
POST /api/auth/refresh

Request:
{
  "token": "eyJhbGciOiJSUzI1NiIs..."
}

Response: (同登录)
```

### 2.3 登出

```
POST /api/auth/logout
Authorization: Bearer <token>

Response:
{
  "code": 0,
  "message": "成功"
}
```

## 三、用户管理接口

### 3.1 患者列表

```
GET /api/user/patients?page=1&pageSize=10&name=张三
Authorization: Bearer <token>

Response: PageData<Patient>
```

### 3.2 科室列表

```
GET /api/user/departments
Authorization: Bearer <token>

Response: Department[]
```

### 3.3 员工列表

```
GET /api/user/employees?page=1&pageSize=10&deptId=dept-001
Authorization: Bearer <token>

Response: PageData<Employee>
```

## 四、挂号预约接口

### 4.1 查询号源

```
GET /api/registration/schedules?deptId=&date=2026-05-10
Authorization: Bearer <token>

Response: Schedule[]
```

### 4.2 挂号

```
POST /api/registration/register
Authorization: Bearer <token>

Request:
{
  "patientId": "uuid",
  "patientName": "张三",
  "scheduleId": "uuid"
}

Response: Registration
```

### 4.3 取消挂号 / 签到

```
POST /api/registration/cancel/:id
POST /api/registration/signin/:id
Authorization: Bearer <token>
```
