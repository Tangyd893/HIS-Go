# HIS-Go 患者端 — 本地快速启动

> 5 分钟内启动小程序 + 患者 H5 演示。

## 前置条件

| 软件 | 版本 | 说明 |
|------|------|------|
| Docker Desktop | 20.10+ | 运行 PostgreSQL / Redis |
| Node.js | 18+ | 患者 H5 Vite 开发服务器 |
| Go | 1.23+ | 后端微服务编译运行 |
| 微信开发者工具 | 最新版 | 导入小程序壳 |

## 一键启动

```bash
# Windows（双击或命令行）
scripts\start-patient.bat

# macOS / Linux
bash scripts/demo-patient.sh start
cd frontend/patient && npm run dev -- --host
```

## 手动分步启动

### 1. 基础设施（无需 RabbitMQ）

```bash
docker compose -f deploy/compose/demo-patient.yml --env-file deploy/config/demo.env up -d postgresql redis
```

### 2. 初始化数据库

```bash
docker exec his-postgres-demo psql -U his_admin -f /sql-init/init_all.sql
docker exec his-postgres-demo psql -U his_admin -f /sql-init/seed_data.sql
```

### 3. 启动后端微服务（患者端 Profile）

```bash
cd backend
# 密码已在 deploy/config/demo.env 中配置（强密码），本地 go run 时需手动设置环境变量
# 参考 deploy/config/demo.env 中的实际密码值
set DB_PASSWORD=<见 demo.env>
set REDIS_PASSWORD=<见 demo.env>

go run ./cmd/gateway &
go run ./cmd/auth &
go run ./cmd/user &
go run ./cmd/registration &
go run ./cmd/schedule &
go run ./cmd/prescription &
go run ./cmd/examination &
go run ./cmd/followup &
go run ./cmd/health_record &
```

### 4. 启动患者 H5

```bash
cd frontend/patient
npm install
npm run dev -- --host
```

浏览器打开 **http://localhost:5174**

### 5. 导入微信开发者工具

1. 打开**微信开发者工具**
2. **导入项目** → 选择 `frontend/mp-webview` 目录
3. 填写 AppID（测试号即可）
4. **详情 → 本地设置** → 勾选 ✅ **不校验合法域名**
5. 编译运行，模拟器中应显示患者端页面

## 就诊助手（AI 分诊，可选）

### 配置 API Key

编辑 `deploy/config/demo.env`（演示环境已预配置 Key，可直接使用）：

```bash
# DeepSeek Chat API（用于生成挂号建议）
DEEPSEEK_API_KEY=sk-your-deepseek-key
# SiliconFlow BGE-M3 嵌入 API（语义检索）
SILICONFLOW_API_KEY=sk-your-siliconflow-key
TRIAGE_RAG_ENABLED=true
```

> 当前 `deploy/config/demo.env` 已包含演示 Key，可直接调用就诊助手 API。

### 构建知识库嵌入

```bash
# 需先配置 SILICONFLOW_API_KEY
make triage-embed
```

若跳过，就诊助手自动降级为**关键词模式**（仍可用）。

### 重启 outpatient 服务

```bash
# Docker 演示
docker compose -f deploy/compose/demo-patient.yml --env-file deploy/config/demo.env up -d --build his-outpatient

# 或 go run 方式：重启 outpatient 窗口使环境变量生效
```

## 就诊助手验证

### 通过 Gateway 调用（推荐）

```bash
# 1. 先登录获取 Token
TOKEN=$(curl -s http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"demo-patient","password":"demo123"}' | \
  python3 -c "import sys,json; print(json.load(sys.stdin)['data']['token'])")

# 2. 调用就诊助手
curl -s http://localhost:8080/api/outpatient/assistant/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"symptom":"咳嗽三天发烧"}'

# 预期响应（LLM 模式）：
# {"code":0,"data":{"mode":"llm","advice":"推荐科室：内科...","depts":[{"id":"dept_001","name":"内科"}],"disclaimer":"..."}}
#
# 若 mode 为 "keyword"：Key 未配置或 API 调用失败，自动降级
```

### 直接调用 outpatient 服务（无需 JWT）

```bash
curl -s -X POST http://localhost:8091/api/outpatient/assistant/chat \
  -H "Content-Type: application/json" \
  -d '{"symptom":"咳嗽三天发烧"}'
```

## 常见问题

| 现象 | 解决 |
|------|------|
| web-view 白屏 | 确认患者 H5 `:5174` 已启动且 `--host` 参数已加 |
| 无法打开 web-view | 详情 → 勾选「不校验合法域名」 |
| 登录失败 | 确认 demo-patient / demo123，检查数据库初始化 |
| 真机无法加载 | 手机和 PC 需同一 WiFi，H5 地址用局域网 IP |
| 处方/报告为空 | 数据库已有种子数据，检查 seed_data.sql 是否正确执行 |
| 就诊助手返回 keyword 模式 | 检查 demo.env 中 DEEPSEEK_API_KEY 和 SILICONFLOW_API_KEY |
| Docker 端口冲突 | demo-admin 和 demo-patient 共用基础设施，先停一个再启另一个 |
