# HIS-Go 患者端 — 本地快速启动

> 5 分钟内启动小程序 + 患者 H5 演示。

## 前置条件

| 软件 | 版本 | 说明 |
|------|------|------|
| Docker Desktop | 20.10+ | 运行 PostgreSQL / Redis |
| Node.js | 18+ | 患者 H5 Vite 开发服务器 |
| Go | 1.23+ | 后端微服务编译运行 |
| 微信开发者工具 | 最新版 | 导入小程序壳 |

## 一键启动

```bash
# Windows（双击或命令行）
scripts\start-patient.bat

# macOS / Linux
bash scripts/demo-patient.sh start
cd frontend/patient && npm run dev -- --host
```

## 手动分步启动

### 1. 基础设施（无需 RabbitMQ）

```bash
docker compose -f deploy/compose/demo-patient.yml --env-file deploy/config/demo.env up -d postgresql redis
```

### 2. 初始化数据库

```bash
docker exec his-postgres-demo psql -U his_admin -f /sql-init/init_all.sql
docker exec his-postgres-demo psql -U his_admin -f /sql-init/seed_data.sql
```

### 3. 启动后端微服务（患者端 Profile）

```bash
cd backend
set DB_PASSWORD=change_me_123
set REDIS_PASSWORD=change_me_456

go run ./cmd/gateway &
go run ./cmd/auth &
go run ./cmd/user &
go run ./cmd/registration &
go run ./cmd/schedule &
go run ./cmd/prescription &
go run ./cmd/examination &
go run ./cmd/followup &
go run ./cmd/health_record &
```

### 4. 启动患者 H5

```bash
cd frontend/patient
npm install
npm run dev -- --host
```

浏览器打开 **http://localhost:5174**

### 5. 导入微信开发者工具

1. 打开**微信开发者工具**
2. **导入项目** → 选择 `frontend/mp-webview` 目录
3. 填写 AppID（测试号即可）
4. **详情 → 本地设置** → 勾选 ✅ **不校验合法域名**
5. 编译运行，模拟器中应显示患者端页面

## 停止

```bash
docker compose -f deploy/compose/demo-patient.yml down
# 关闭 go run 窗口 + Ctrl+C 前端
```

## 真机预览（可选）

1. 手机与 PC 连接同一 WiFi
2. 修改 `frontend/mp-webview/pages/index/index.js`：
   ```javascript
   const DEV_PATIENT_H5 = 'http://192.168.x.x:5174'  // PC 局域网 IP
   ```
3. 微信开发者工具 → **预览** → 手机扫码

## 常见问题

| 现象 | 解决 |
|------|------|
| web-view 白屏 | 确认患者 H5 `:5174` 已启动且 `--host` 参数已加 |
| 无法打开 web-view | 详情 → 勾选「不校验合法域名」 |
| 登录失败 | 重新执行步骤 2 初始化数据库 |
| 真机无法加载 | 手机和 PC 需同一 WiFi，H5 地址用局域网 IP |
| 处方/报告为空 | 数据库已有种子数据，检查步骤 2 是否正确执行 |
