# HIS-Go 演示验证流程

## 概述

本文档详细说明了 HIS-Go 演示环境的验证流程，包括验证步骤、验证方法和验证标准。

## 验证流程

### 1. 环境准备

#### 1.1 检查前置条件

```bash
# 检查 Docker 版本
docker --version

# 检查 Docker Compose 版本
docker compose version

# 检查系统资源
free -h
df -h
```

#### 1.2 准备环境变量

```bash
# 复制环境变量模板
cp docker/.env.demo.example docker/.env.demo

# 编辑环境变量
vim docker/.env.demo
```

### 2. 启动演示服务

#### 2.1 启动管理端演示

```bash
# 使用 Makefile
make demo-admin

# 或使用脚本
./scripts/demo-admin.sh start

# 或使用 Docker Compose
docker compose -f docker/docker-compose.demo-admin.yml --env-file docker/.env.demo up -d
```

#### 2.2 启动患者端演示

```bash
# 使用 Makefile
make demo-patient

# 或使用脚本
./scripts/demo-patient.sh start

# 或使用 Docker Compose
docker compose -f docker/docker-compose.demo-patient.yml --env-file docker/.env.demo up -d
```

### 3. 验证服务状态

#### 3.1 检查容器状态

```bash
# 查看所有容器
docker ps

# 查看演示服务状态
make demo-status

# 查看特定服务状态
docker compose -f docker/docker-compose.demo-admin.yml ps
```

#### 3.2 检查服务健康

```bash
# 检查 Gateway 健康状态
curl http://localhost:8080/health

# 检查各服务健康状态
curl http://localhost:8081/health  # Auth
curl http://localhost:8082/health  # User
curl http://localhost:8083/health  # Registration
curl http://localhost:8084/health  # Clinic
curl http://localhost:8085/health  # Prescription
curl http://localhost:8086/health  # Billing
curl http://localhost:8087/health  # Pharmacy
curl http://localhost:8090/health  # Schedule
curl http://localhost:8096/health  # System
```

#### 3.3 检查端口占用

```bash
# 检查端口占用
netstat -tulpn | grep :8080
netstat -tulpn | grep :5432
netstat -tulpn | grep :6379
```

### 4. 验证前端访问

#### 4.1 检查 Nginx 配置

```bash
# 检查 Nginx 配置
docker compose -f docker/docker-compose.demo-admin.yml exec nginx nginx -t

# 查看 Nginx 日志
docker compose -f docker/docker-compose.demo-admin.yml logs nginx
```

#### 4.2 检查前端访问

```bash
# 检查管理端前端
curl -I http://localhost/admin

# 检查患者端前端
curl -I http://localhost/patient

# 检查根路径重定向
curl -I http://localhost/
```

### 5. 验证 API 接口

#### 5.1 测试登录接口

```bash
# 管理端登录
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"demo-admin","password":"demo123"}'

# 患者端登录
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"demo-patient","password":"demo123"}'
```

#### 5.2 测试用户信息接口

```bash
# 获取 Token
TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"demo-admin","password":"demo123"}' | jq -r '.data.token')

# 获取当前用户信息
curl -H "Authorization: Bearer $TOKEN" http://localhost:8080/api/auth/current
```

#### 5.3 测试其他接口

```bash
# 测试排班列表
curl -H "Authorization: Bearer $TOKEN" http://localhost:8080/api/schedule/list

# 测试挂号列表
curl -H "Authorization: Bearer $TOKEN" http://localhost:8080/api/registration/list

# 测试处方列表
curl -H "Authorization: Bearer $TOKEN" http://localhost:8080/api/prescription/list
```

### 6. 验证功能流程

#### 6.1 管理端主路径流程

1. **登录系统**
   - 访问 http://localhost/admin
   - 使用 demo-admin / demo123 登录
   - 验证登录成功

2. **查看排班**
   - 进入排班管理页面
   - 查看排班列表
   - 验证数据加载正常

3. **挂号操作**
   - 进入挂号管理页面
   - 为患者挂号
   - 验证挂号成功

4. **门诊接诊**
   - 进入门诊诊疗页面
   - 录入诊断信息
   - 验证接诊成功

5. **开具处方**
   - 进入处方管理页面
   - 开具处方
   - 验证处方创建成功

6. **收费操作**
   - 进入收费管理页面
   - 完成收费
   - 验证收费成功

7. **药房发药**
   - 进入药房管理页面
   - 确认发药
   - 验证发药成功

#### 6.2 患者端主路径流程

1. **登录系统**
   - 访问 http://localhost/patient
   - 使用患者账号登录
   - 验证登录成功

2. **预约挂号**
   - 进入预约挂号页面
   - 选择科室和医生
   - 选择时间段
   - 验证预约成功

3. **查看处方**
   - 进入我的处方页面
   - 查看处方列表
   - 验证数据加载正常

4. **检查报告**
   - 进入检查报告页面
   - 查看检查结果
   - 验证数据加载正常

5. **随访任务**
   - 进入随访页面
   - 查看随访任务列表
   - 验证数据加载正常

### 7. 验证小程序

#### 7.1 导入小程序

1. 打开微信开发者工具
2. 选择"导入项目"
3. 项目目录选择 `frontend/his-mp-webview`
4. AppID 使用测试号或真实 AppID

#### 7.2 配置 H5 地址

编辑 `frontend/his-mp-webview/pages/index/index.js`：

```javascript
// 开发环境
const DEFAULT_PATIENT_H5 = 'http://localhost:5174'

// 云服务器环境
// const DEFAULT_PATIENT_H5 = 'https://your-domain.com/patient/'
```

#### 7.3 验证小程序

1. 编译小程序
2. 点击"预览"或"真机调试"
3. 使用手机微信扫描二维码
4. 验证 web-view 加载 H5 正常
5. 验证登录功能正常
6. 验证主路径流程正常

### 8. 性能验证

#### 8.1 资源监控

```bash
# 查看容器资源使用
docker stats

# 查看系统资源
htop

# 查看磁盘使用
df -h
```

#### 8.2 响应时间测试

```bash
# 安装 Apache Bench
sudo apt install apache2-utils

# 测试 Gateway 性能
ab -n 100 -c 10 http://localhost:8080/health

# 测试登录接口性能
ab -n 100 -c 10 -p login.json -T application/json http://localhost:8080/api/auth/login
```

### 9. 安全验证

#### 9.1 JWT 认证验证

```bash
# 测试无 Token 访问
curl http://localhost:8080/api/auth/current

# 测试无效 Token 访问
curl -H "Authorization: Bearer invalid_token" http://localhost:8080/api/auth/current

# 测试过期 Token 访问
curl -H "Authorization: Bearer expired_token" http://localhost:8080/api/auth/current
```

#### 9.2 权限控制验证

```bash
# 测试普通用户访问管理接口
curl -H "Authorization: Bearer $PATIENT_TOKEN" http://localhost:8080/api/system/list
```

### 10. 稳定性验证

#### 10.1 长时间运行测试

```bash
# 启动服务
make demo-admin

# 等待一段时间
sleep 3600

# 检查服务状态
make demo-status

# 检查日志
make demo-admin-logs
```

#### 10.2 异常恢复测试

```bash
# 停止某个服务
docker compose -f docker/docker-compose.demo-admin.yml stop his-auth

# 等待一段时间
sleep 60

# 重新启动服务
docker compose -f docker/docker-compose.demo-admin.yml start his-auth

# 检查服务状态
make demo-status
```

## 验证标准

### 1. 服务启动标准

- 所有容器正常运行
- 所有服务健康检查通过
- 无异常日志输出

### 2. 功能验证标准

- 所有页面正常加载
- 所有 API 接口正常响应
- 主路径流程正常完成

### 3. 性能验证标准

- 服务响应时间 < 500ms
- 资源使用合理
- 无内存泄漏

### 4. 安全验证标准

- JWT 认证正常
- 权限控制正常
- 敏感信息未泄露

### 5. 稳定性验证标准

- 服务无异常重启
- 日志无错误信息
- 数据库连接正常
