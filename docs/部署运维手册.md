# HIS-Go 部署运维手册

## 一、环境要求

| 组件 | 最低版本 | 说明 |
|------|---------|------|
| Docker | 20.10+ | 容器运行时 |
| Docker Compose | 2.0+ | 容器编排 |
| Go | 1.25+ | 仅本地开发需要 |
| Node.js | 24 LTS | 仅前端构建需要 |

## 二、快速部署

### 2.1 配置环境变量

```bash
cd docker
cp .env.example .env
# 编辑 .env 文件，设置数据库密码、Redis密码、JWT密钥等
```

必需的环境变量：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| DB_PASSWORD | 数据库密码 | change_me_123 |
| REDIS_PASSWORD | Redis 密码 | change_me_456 |
| MQ_PASSWORD | RabbitMQ 密码 | change_me_789 |
| MINIO_SECRET_KEY | MinIO 密钥 | change_me_012 |
| JWT_PRIVATE_KEY | JWT 私钥 (PEM) | 自动生成 HS256 |
| JWT_PUBLIC_KEY | JWT 公钥 (PEM) | 自动生成 HS256 |

### 2.2 生成 JWT 密钥对

```bash
openssl genrsa -out private.pem 2048
openssl rsa -in private.pem -pubout -out public.pem

# 将 PEM 内容写入 .env
JWT_PRIVATE_KEY=$(cat private.pem | base64 -w0)
JWT_PUBLIC_KEY=$(cat public.pem | base64 -w0)
```

### 2.3 启动全部服务

```bash
# 开发环境
docker compose -f docker/docker-compose.yml up -d

# 生产环境
docker compose -f docker/docker-compose.prod.yml up -d
```

### 2.4 验证部署

```bash
# 健康检查
curl http://localhost:8080/health
curl http://localhost:8080/ready

# 登录测试
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"demo-admin","password":"demo123"}'

# 带 Token 访问受保护接口
curl http://localhost:8080/api/user/departments \
  -H "Authorization: Bearer <token>"
```

## 三、本地开发

### 3.1 启动基础设施

```bash
docker compose -f docker/docker-compose.yml up -d postgresql redis rabbitmq nacos minio
```

### 3.2 初始化数据库

```bash
cd backend
bash scripts/db_init.sh
```

### 3.3 启动各微服务

```bash
# 分别在不同终端启动
cd backend
go run ./cmd/gateway &
go run ./cmd/auth &
go run ./cmd/user &
# ... 其他服务
```

## 七、常用运维操作

### 7.1 查看服务日志

```bash
docker compose -f docker/docker-compose.yml logs -f gateway
docker compose -f docker/docker-compose.yml logs -f --tail=100 auth
```

### 7.2 重启单个服务

```bash
docker compose -f docker/docker-compose.yml restart auth
```

### 7.3 数据库备份

```bash
docker exec his-postgres pg_dumpall -U his_admin > backup_$(date +%Y%m%d).sql
```

### 7.4 数据库恢复

```bash
docker exec -i his-postgres psql -U his_admin < backup.sql
```

### 7.5 清理与重置

```bash
# 停止所有服务
docker compose -f docker/docker-compose.yml down

# 删除所有数据卷（慎用！）
docker compose -f docker/docker-compose.yml down -v

# 重新构建镜像
docker compose -f docker/docker-compose.yml build --no-cache
```

## 八、故障排查

### 8.1 服务无法启动

1. 检查基础设施容器状态：`docker ps -a`
2. 查看容器日志：`docker logs his-gateway`
3. 验证数据库连接：`docker exec his-postgres psql -U his_admin -c "\l"`
4. 检查端口占用：`lsof -i :8080`

### 8.2 数据库连接失败

```bash
# 检查 PostgreSQL 是否就绪
docker exec his-postgres pg_isready -U his_admin

# 查看数据库列表
docker exec his-postgres psql -U his_admin -c "\l"
```

### 8.3 JWT 认证失败

- 检查 JWT 密钥是否已设置环境变量
- 验证 Token 是否过期（默认 2 小时）
- 查看 Redis 中 Token 是否被主动失效

## 九、性能调优

- PostgreSQL：调整 `max_connections`、`shared_buffers`
- Redis：配置 `maxmemory` 淘汰策略
- Go 服务：通过 `GOMAXPROCS` 设置并发度
- Nginx：调整 `worker_processes` 和 `worker_connections`
