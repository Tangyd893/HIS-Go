# HIS-Go 管理端 — 本地快速启动

> 5 分钟内启动 Docker Compose + Vue 管理端演示。

## 前置条件

| 软件 | 版本 | 说明 |
|------|------|------|
| Docker Desktop | 20.10+ | 运行全部微服务（PostgreSQL / Redis / RabbitMQ） |
| Node.js | 18+ | 构建 Vue 管理端 |
| Go | 1.23+ | 后端编译（Docker 内完成，宿主机可选） |

## 一键启动

```bash
# 首次使用：构建前端 + 启动全部服务
cd frontend/admin && npm ci && npm run build && cd ../..
docker compose -f deploy/compose/demo-admin.yml --env-file deploy/config/demo.env up -d --build

# 后续使用（仅重启，无需重建）
docker compose -f deploy/compose/demo-admin.yml --env-file deploy/config/demo.env up -d
```

## 手动分步启动

### 1. 构建前端

```bash
cd frontend/admin
npm ci
npm run build
cd ../..
```

### 2. 配置环境变量

编辑 `deploy/config/demo.env`（已提供，如需自定义密码可修改）：

```bash
POSTGRES_USER=his_admin
POSTGRES_PASSWORD=<your_password>
REDIS_PASSWORD=<your_password>
RABBITMQ_PASSWORD=<your_password>
```

### 3. 启动服务

```bash
docker compose -f deploy/compose/demo-admin.yml --env-file deploy/config/demo.env up -d --build
```

首次构建约需 3-5 分钟（Go 编译 17 个微服务）。

### 4. 访问管理端

浏览器打开 **http://localhost/admin**

## 管理端主路径演示（7 步）

按以下路径操作可覆盖核心业务闭环：

### 1. 排班管理 → 生成排班

菜单「排班管理」→ 点击「生成排班」→ 选择日期范围和科室 → 确认。

生成后表格显示**总号源**和**已约**数量。

### 2. 号源管理 → 患者挂号

菜单「号源管理」→ 选择日期 → 查看号源列表 → 点击「挂号」→ **搜索选择患者**（输入姓名即可搜索）→ 确认挂号。

> 挂号后号源「剩余号源」自动扣减。

### 3. 签到

挂号记录表格中，待签到（蓝色标签）的记录点击「签到」按钮。

### 4. 开始接诊

签到后，挂号记录出现「开始接诊」按钮 → 点击跳转到门诊诊疗页。

> 页面自动填入患者信息，录入主诉和诊断后保存。

### 5. 开具处方

菜单「处方管理」→「开具处方」→ 搜索选择患者 → **添加药品明细**（搜索药品名称、填入数量/用量/用法）→ 保存。

处方初始状态为「待审核」。

### 6. 审核处方

处方列表中点击「审核」→ 处方变为「已审核」状态。

### 7. 收费结算 → 发药

菜单「收费结算」→ 查看待支付账单 → 点击「收费」→ 账单状态变为「已支付」。

菜单「药品管理」→ 选择药品 →「发药」→ **下拉选择已审核的处方** → 填入数量 → 确认发药。

## 常见问题

**Q: 前端修改后不生效？**

重新构建：`npm --prefix frontend/admin run build`。Nginx 使用 bind mount，文件变更即时生效，刷新浏览器即可。

**Q: 后端修改后不生效？**

重新构建镜像并重启：
```bash
docker compose -f deploy/compose/demo-admin.yml --env-file deploy/config/demo.env up -d --build <服务名>
```

**Q: 数据库需要重置？**

```bash
docker compose -f deploy/compose/demo-admin.yml --env-file deploy/config/demo.env down -v
docker compose -f deploy/compose/demo-admin.yml --env-file deploy/config/demo.env up -d --build
```
`-v` 会删除数据卷，重启后 migrations 自动建表，`db-init` 容器自动导入 seed 数据。

**Q: seed 数据未导入？**

手动执行：
```bash
docker exec his-postgres-demo psql -U his_admin -f /sql-init/seed_data.sql
docker exec his-postgres-demo psql -U his_admin -f /sql-init/seed_data_extended.sql
```

**Q: 患者搜索不到？**

患者列表 API 支持按姓名模糊搜索，输入 1 个字符即可触发。确保 `demo-admin.yml` 中 `his-user` 服务正常运行。
