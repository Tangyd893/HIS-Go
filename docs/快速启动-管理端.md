# HIS-Go 管理端 — 本地快速启动

> 5 分钟内从零启动 Web 医院信息管理系统演示。

## 前置条件

| 软件 | 版本 | 说明 |
|------|------|------|
| Docker Desktop | 20.10+ | 运行 PostgreSQL / Redis / RabbitMQ |
| Node.js | 18+ | 前端 Vite 开发服务器 |
| Go | 1.23+ | 后端微服务编译运行 |

## 一键启动

```bash
# Windows（双击或命令行）
scripts\start-admin.bat

# macOS / Linux
bash scripts/demo-admin.sh start
cd frontend/admin && npm run dev
```

## 手动分步启动

### 1. 基础设施

```bash
docker compose -f deploy/compose/demo-admin.yml --env-file deploy/config/demo.env up -d postgresql redis rabbitmq
```

### 2. 初始化数据库

```bash
docker exec his-postgres-demo psql -U his_admin -f /sql-init/init_all.sql
docker exec his-postgres-demo psql -U his_admin -f /sql-init/seed_data.sql
```

### 3. 启动后端微服务

```bash
cd backend
set DB_PASSWORD=change_me_123
set REDIS_PASSWORD=change_me_456
set RABBITMQ_PASSWORD=change_me_789

go run ./cmd/gateway &
go run ./cmd/auth &
go run ./cmd/user &
go run ./cmd/registration &
go run ./cmd/schedule &
go run ./cmd/clinic &
go run ./cmd/prescription &
go run ./cmd/billing &
go run ./cmd/pharmacy &
go run ./cmd/system &
```

### 4. 启动前端

```bash
cd frontend/admin
npm install
npm run dev
```

浏览器打开 **http://localhost:5173**

## 停止

```bash
docker compose -f deploy/compose/demo-admin.yml down
# 关闭各个 go run 窗口或 Ctrl+C 前端
```

## 常见问题

| 现象 | 解决 |
|------|------|
| `docker compose` 报错 | 确认 Docker Desktop 正在运行 |
| 端口 5432 冲突 | 已映射为 5433，如有其他冲突检查 `netstat -an` |
| 前端白屏 | 确认 Gateway `:8080` 已启动：`curl localhost:8080/health` |
| 登录失败 | 重新执行步骤 2 初始化数据库 |
| Go 依赖下载慢 | 设置代理：`set GOPROXY=https://goproxy.cn,direct` |
