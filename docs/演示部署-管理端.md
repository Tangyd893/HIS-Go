# HIS-Go 管理端演示部署指南

## 概述

本文档说明如何部署 HIS-Go 管理端演示环境，包括本地开发环境和云服务器部署。

## 前置条件

- Docker 20.10+
- Docker Compose v2.0+
- Node.js 18+ (仅构建前端时需要)
- Go 1.21+ (仅本地开发时需要)

## 快速开始

### 1. 环境准备

```bash
# 克隆项目
git clone <repository-url>
cd HIS-Go

# 复制环境变量模板
cp docker/.env.demo.example docker/.env.demo

# 根据需要修改环境变量
vim docker/.env.demo
```

### 2. 构建前端 (可选)

```bash
# 构建管理端前端
make build-admin-react

# 或者使用 npm
cd frontend/his-web-admin-react
npm ci --legacy-peer-deps
npm run build
```

### 3. 启动演示服务

#### 方式一：使用 Makefile

```bash
# 启动管理端演示
make demo-admin

# 查看日志
make demo-admin-logs

# 停止服务
make demo-admin-stop
```

#### 方式二：使用脚本

```bash
# 启动服务
./scripts/demo-admin.sh start

# 查看服务状态
./scripts/demo-admin.sh status

# 查看日志
./scripts/demo-admin.sh logs

# 停止服务
./scripts/demo-admin.sh stop
```

#### 方式三：直接使用 Docker Compose

```bash
# 启动服务
docker compose -f docker/docker-compose.demo-admin.yml --env-file docker/.env.demo up -d

# 查看服务状态
docker compose -f docker/docker-compose.demo-admin.yml ps

# 查看日志
docker compose -f docker/docker-compose.demo-admin.yml logs -f

# 停止服务
docker compose -f docker/docker-compose.demo-admin.yml down
```

```bash

```

## 云服务器部署

### 1. 服务器准备

```bash
# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 配置防火墙
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

### 2. 部署应用

```bash
# 克隆代码
git clone <repository-url>
cd HIS-Go

# 配置环境变量
cp docker/.env.demo.example docker/.env.demo
vim docker/.env.demo

# 构建前端
make build-admin-react

# 启动服务
make demo-admin
```

### 3. 配置 HTTPS (推荐)

```bash
# 安装 Certbot
sudo apt install certbot

# 获取证书
sudo certbot certonly --standalone -d your-domain.com

# 配置 Nginx SSL (需要修改 nginx.conf)
```

## 性能优化

### 资源限制

在 `docker-compose.demo-admin.yml` 中添加资源限制：

```yaml
services:
  his-auth:
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 512M
        reservations:
          cpus: '0.25'
          memory: 256M
```

### 数据库优化

```bash
# 调整 PostgreSQL 配置
# 在 docker-compose.yml 中添加 volumes
volumes:
  - ./postgresql.conf:/etc/postgresql/postgresql.conf
```

## 备份与恢复

### 数据库备份

```bash
# 备份数据库
docker compose -f docker/docker-compose.demo-admin.yml exec postgresql pg_dump -U his_admin his_auth > backup.sql

# 恢复数据库
docker compose -f docker/docker-compose.demo-admin.yml exec -T postgresql psql -U his_admin his_auth < backup.sql
```

## 常见问题

### Q: 如何重置演示数据？

```bash
# 停止服务
./scripts/demo-admin.sh stop

# 删除数据卷
docker volume rm his-go_postgres_data

# 重新启动
./scripts/demo-admin.sh start
```

### Q: 如何修改演示账号密码？

1. 修改 `backend/sql/seed_data.sql` 中的密码哈希
2. 重新构建并启动服务

### Q: 如何添加新的微服务？

1. 在 `backend/cmd/` 下创建新服务
2. 在 `docker/docker-compose.demo-admin.yml` 中添加服务配置
3. 在 `docker/nginx/nginx.conf` 中添加路由
4. 更新 Gateway 路由表
