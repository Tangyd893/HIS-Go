# HIS-Go 演示验证工具使用示例

## 概述

本文档提供了 HIS-Go 演示验证工具的使用示例，帮助用户快速上手使用验证工具。

## 快速开始

### 1. 验证管理端演示

```bash
# 启动管理端演示服务
make demo-admin

# 等待服务启动完成
sleep 30

# 验证管理端演示环境
make verify-admin
```

### 2. 验证患者端演示

```bash
# 启动患者端演示服务
make demo-patient

# 等待服务启动完成
sleep 30

# 验证患者端演示环境
make verify-patient
```

### 3. 测试 API 接口

```bash
# 确保演示服务已启动
make demo-admin

# 测试 API 接口
make demo-verify
```

### 4. 验证所有演示环境

```bash
# 启动管理端演示服务
make demo-admin

# 等待服务启动完成
sleep 30

# 验证所有演示环境
make demo-verify
```

## 自定义验证

### 1. 编辑验证脚本

```bash
vim scripts/demo-verify.sh
```

### 2. 添加自定义验证函数

在脚本中添加：

```bash
# 验证自定义功能
verify_custom_feature() {
    print_info "验证自定义功能..."
    
    # 添加验证逻辑
    
    if [ 验证通过 ]; then
        print_success "新功能验证通过"
        return 0
    else
        print_error "新功能验证失败"
        return 1
    fi
}
```

### 3. 在主函数中添加自定义命令

在 `main()` 函数中添加：

```bash
main() {
    case "${1:-help}" in
        # ... 其他命令 ...
        custom)
            verify_custom_feature
            ;;
        # ... 其他命令 ...
    esac
}
```

### 4. 运行自定义验证

```bash
./scripts/demo-verify.sh custom
```

## 故障排查

### 1. 服务启动失败

```bash
# 查看服务日志
make demo-admin-logs

# 查看特定服务日志
./scripts/demo-admin.sh logs his-auth

# 检查 Docker 状态
docker ps -a
```

### 2. 健康检查失败

```bash
# 检查服务状态
docker ps

# 查看服务日志
./scripts/demo-admin.sh logs his-auth

# 检查端口占用
netstat -tulpn | grep :8081
```

### 3. 前端访问失败

```bash
# 检查 Nginx 状态
docker compose -f docker/docker-compose.demo-admin.yml ps nginx

# 查看 Nginx 日志
docker compose -f docker/docker-compose.demo-admin.yml logs nginx

# 检查前端构建产物
ls -la frontend/his-web-admin-react/dist/
```

### 4. API 接口测试失败

```bash
# 检查 Gateway 状态
curl http://localhost:8080/health

# 测试登录接口
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"demo-admin","password":"demo123"}'

# 查看 Gateway 日志
./scripts/demo-admin.sh logs his-gateway
```
