# HIS-Go 云端部署操作指南

> 版本：v1.0  
> 日期：2026-06-03  
> 场景：**单台云服务器演示部署**（非生产 HA 集群）

---

## 1. 部署什么

### 1.1 两个演示前端

| 场景 | 前端 | 目录 | 访问路径 |
|------|------|------|----------|
| **Web 管理端** | Vue 3 + Ant Design Vue | `frontend/admin` | `https://域名/admin` |
| **小程序患者端** | 小程序壳 + Vue H5 | `frontend/mp-webview` → `frontend/patient` | `https://域名/patient` |

小程序不在服务器上运行，只在微信开发者工具/真机中打开；服务器托管的是 **患者 H5**（`patient`）和 **API**。

React 版前端已移至 `frontend/archive/`，**不参与演示部署**。

### 1.2 演示时调用的后端微服务

通过 **Gateway** 统一入口 `/api/*`，按 Profile **只启动演示需要的子集**：

#### Profile `demo-admin`（管理端）

| 微服务 | 端口 | 用途 |
|--------|------|------|
| gateway | 8080 | 统一 API 入口 |
| auth | 8081 | 登录 |
| user | 8082 | 患者/员工/科室 |
| registration | 8083 | 挂号 |
| schedule | 8090 | 排班 |
| clinic | 8084 | 门诊接诊 |
| prescription | 8085 | 处方 |
| billing | 8086 | 收费 |
| pharmacy | 8087 | 药房 |
| system | 8096 | 字典/参数 |

**主路径**：登录 → 排班 → 挂号 → 接诊 → 处方 → 收费 → 发药

#### Profile `demo-patient`（患者端 / 小程序）

| 微服务 | 端口 | 用途 |
|--------|------|------|
| gateway | 8080 | 统一 API 入口 |
| auth | 8081 | 登录 |
| user | 8082 | 用户信息 |
| registration | 8083 | 预约挂号 |
| schedule | 8090 | 号源 |
| prescription | 8085 | 我的处方 |
| examination | 8088 | 检查报告 |
| followup | 8092 | 随访 |
| health_record | 8093 | 健康档案 |

**主路径**：登录 → 预约 → 处方 → 检查报告

> 可选扩展：+ `outpatient`（8091）在线问诊/慢病。  
> **不启动**：clinic、billing、pharmacy、inpatient、emr、statistics 等管理专用服务。

### 1.3 中间件能否用 Docker 管理？

**可以，且推荐全部用 Docker Compose 管理。**

| 中间件 | demo-admin | demo-patient | 说明 |
|--------|:----------:|:------------:|------|
| **PostgreSQL 17** | ✅ | ✅ | 数据持久化，Compose volume |
| **Redis 7** | ✅ | ✅ | 各微服务启动依赖 |
| **RabbitMQ 4** | ✅ | ❌ | 仅 billing/pharmacy 需要 |
| **Nginx** | ✅ | ✅ | 静态资源 + `/api` 反代 |
| Nacos | ❌ | ❌ | 演示用 Gateway 静态路由 |
| MinIO | ❌ | ❌ | 演示未接入 |

配置文件位置：

```
deploy/
├── compose/
│   ├── demo-admin.yml      # 管理端 Profile（含 PG/Redis/MQ/Nginx/微服务）
│   └── demo-patient.yml    # 患者端 Profile（含 PG/Redis/Nginx/微服务，无 MQ）
├── nginx/nginx.conf
├── config/
│   ├── demo.env.example    # 演示环境变量模板
│   └── demo.env            # 实际配置（勿提交 Git）
└── Dockerfile              # Go 微服务镜像构建
```

## 2. 上云前本地验证（必做）

```bash
# 1. 配置环境变量
cp deploy/config/demo.env.example deploy/config/demo.env
# 编辑 deploy/config/demo.env，修改密码

# 2. 构建 Vue 前端
make build-frontend

# 3. 启动并验证（二选一）
make demo-admin    # → http://localhost/admin  （demo-doctor / demo123）
make demo-patient  # → http://localhost/patient（demo-patient / demo123）

# 4. 健康检查
curl http://localhost:8080/health
```

本地未跑通勿上云。

## 3. 云服务器要求

| 项 | 建议 |
|----|------|
| 规格 | 2 vCPU / 4 GB RAM / 40 GB SSD（任一 Profile） |
| 系统 | Ubuntu 22.04 / 24.04 LTS |
| 软件 | Docker 24+、Docker Compose v2、Node.js 20+（构建前端） |
| 域名 | 已备案（小程序真机 HTTPS 必须） |
| 安全组 | **仅开放 22、80、443**；禁止公网暴露 5432/6379/8080 |

一键初始化 VPS（root/sudo）：

```bash
sudo bash scripts/cloud-init.sh
```

## 4. 部署步骤

### 4.1 获取代码

```bash
git clone <仓库地址> HIS-Go && cd HIS-Go
```

### 4.2 配置 secrets

```bash
cp deploy/config/demo.env.example deploy/config/demo.env
vim deploy/config/demo.env
```

至少修改：

```bash
POSTGRES_PASSWORD=<强密码>
REDIS_PASSWORD=<强密码>
RABBITMQ_PASSWORD=<强密码>   # 仅 demo-admin
```

### 4.3 构建前端

```bash
make build-frontend
# 产物：frontend/admin/dist、frontend/patient/dist
```

### 4.4 启动 Profile

**管理端演示：**

```bash
docker compose -f deploy/compose/demo-admin.yml --env-file deploy/config/demo.env up -d --build
```

**患者端 / 小程序演示：**

```bash
docker compose -f deploy/compose/demo-patient.yml --env-file deploy/config/demo.env up -d --build
```

或使用 Makefile：

```bash
make demo-admin
# 或
make demo-patient
```

### 4.5 验证

```bash
curl http://127.0.0.1:8080/health
curl -I http://127.0.0.1/admin      # 管理端
curl -I http://127.0.0.1/patient    # 患者端
docker compose -f deploy/compose/demo-admin.yml ps
```

## 5. HTTPS（小程序真机必须）

```bash
# 临时停 Nginx 占用的 80 端口
docker compose -f deploy/compose/demo-patient.yml stop nginx

sudo apt install -y certbot
sudo certbot certonly --standalone -d your-domain.com
```

在 `deploy/nginx/` 增加 SSL 配置，挂载 `/etc/letsencrypt`，Nginx 监听 443。示例见本文档历史版本或自行扩展 `nginx.conf`。

小程序配置 `frontend/mp-webview/pages/index/index.js`：

```javascript
const CLOUD_PATIENT_H5 = 'https://your-domain.com/patient/'
const USE_CLOUD = true
```

微信公众平台 → **业务域名** 添加同域名。

## 6. 同机切换 Profile

两个 Profile **不要同时 up**（端口与资源冲突）。

```bash
# 管理端 → 患者端
docker compose -f deploy/compose/demo-admin.yml down
docker compose -f deploy/compose/demo-patient.yml up -d

# 患者端 → 管理端
docker compose -f deploy/compose/demo-patient.yml down
docker compose -f deploy/compose/demo-admin.yml up -d
```

PostgreSQL 数据卷可保留，演示数据在切换后仍在。

## 9. 常见问题

| 现象 | 处理 |
|------|------|
| `/admin` 白屏 | 确认 `make build-admin` 且 Compose 挂载 `frontend/admin/dist` |
| API 401 | 检查 seed 是否导入；查看 auth/gateway 日志 |
| billing 启动失败 | demo-admin 须包含 RabbitMQ 容器 |
| 小程序真机白屏 | HTTPS + 业务域名；H5 须公网可访问 |
| 内存 OOM | 只跑一个 Profile；升级至 4 GB+ |
