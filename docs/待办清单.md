## 二、复用 `his-mp-webview` 可行性评估

### 2.1 结论：**可行，改动量极小**

[HIS-Mixed-Go-Java 的 `his-mp-webview`](https://github.com/Tangyd893/HIS-Mixed-Go-Java/tree/main/frontend/his-mp-webview) **不是**独立业务小程序，而是：

- 微信小程序 **演示壳**（约 6 个文件）
- 仅含一个页面，通过 `<web-view>` 加载已有 **患者端 H5**
- **不包含**挂号、登录等业务逻辑；业务全在 H5 + 后端

因此复用到 HIS-Go **不涉及后端改造**，只需：

1. 将目录拷贝到 `frontend/his-mp-webview`
2. 修改 H5 加载地址（端口 / 部署路径）
3. 确保被加载的患者 H5 可访问且 API 可达

### 2.2 与 HIS-Go 的差异对照

| 项 | HIS-Mixed-Go-Java | HIS-Go（当前） | 复用时需调整 |
|----|-------------------|----------------|--------------|
| 被加载 H5 | `his-web-patient`（Vue） | `his-web-patient-react`（React） | 改 `DEFAULT_PATIENT_H5` 地址 |
| 患者 H5 默认端口 | 5174 | vite 配置 5173（与管理端冲突） | 患者端改为 **5174** |
| 后端 | Go + Java 混合 | 纯 Go 18 服务 | 无小程序侧改动；确保 Gateway :8080 |
| 生产部署 | 未在壳内体现 | Nginx `/patient` | 生产 URL 改为 `https://域名/patient/` |

### 2.3 微信 web-view 限制（必须知晓）

| 场景 | 要求 |
|------|------|
| 本地开发者工具演示 | 勾选「不校验合法域名、web-view（业务域名）、TLS…」 |
| 真机预览 | H5 地址不能用 `127.0.0.1`，须改为 PC **局域网 IP**；H5 需 `npm run dev -- --host` |
| 正式上线 | H5 域名须 **HTTPS** 且在微信公众平台配置 **业务域名**；小程序需正式/测试 AppID |

## 三、复用方案（具体操作步骤）

### 3.1 目录引入

```bash
# 在 HIS-Go 根目录执行
mkdir -p frontend/his-mp-webview
# 方式 A：从上游仓库拷贝（推荐保留 git 溯源说明）
curl -L "https://github.com/Tangyd893/HIS-Mixed-Go-Java/archive/refs/heads/main.zip" -o /tmp/his-mixed.zip
# 解压后仅复制 frontend/his-mp-webview/* → frontend/his-mp-webview/

# 方式 B：git sparse-checkout 或手动复制以下文件：
#   app.js, app.json, app.wxss, project.config.json
#   pages/index/index.js, index.wxml, index.wxss, index.json
```

建议在 `frontend/his-mp-webview/README.md` 中注明来源与 HIS-Go 适配说明（替换原 Mixed 项目中的 Java 启动步骤）。

### 3.2 修改小程序壳（必做）

**文件：`pages/index/index.js`**

```javascript
// 开发：模拟器
const DEFAULT_PATIENT_H5 = 'http://127.0.0.1:5174'

// 真机预览：改为 PC 局域网 IP，例如
// const DEFAULT_PATIENT_H5 = 'http://192.168.1.100:5174'

// 生产（Nginx 已部署患者端静态资源 + HTTPS 业务域名已配置）
// const DEFAULT_PATIENT_H5 = 'https://his.example.com/patient/'
```

**文件：`project.config.json`**

- 确认 `"urlCheck": false`（仅本地演示）
- 替换 `appid`：演示可用 `touristappid`；真机受限时改用微信测试号 AppID

### 3.3 修改 HIS-Go 患者端 H5（必做）

**① 开发端口与管理端分离**

`frontend/his-web-patient-react/vite.config.ts`：

```typescript
server: {
  port: 5174,  // 与管理端 5173 区分
  host: true,  // 真机预览需要
  proxy: { /* 保持不变 */ },
},
```

同步更新 `README.md` 中患者端端口说明（与 vite 配置一致）。

**② 生产部署 base 路径（Nginx `/patient`）**

`vite.config.ts`：

```typescript
export default defineConfig({
  base: '/patient/',
  // ...
})
```

`src/main.tsx`：

```typescript
<BrowserRouter basename="/patient">
```

重新 `npm run build` 后挂载到 Nginx `html/patient`。

### 3.4 联调启动顺序

```bash
# 1. 基础设施 + 后端
cd docker && docker compose up -d
cd ../backend && bash scripts/db_init.sh

# 2. 患者 H5（被 web-view 加载的对象）
cd frontend/his-web-patient-react
npm install --legacy-peer-deps
npm run dev -- --host    # → http://127.0.0.1:5174

# 3. 微信开发者工具
#    导入 frontend/his-mp-webview
#    详情 → 本地设置 → 勾选不校验合法域名
#    编译运行
```

### 3.5 生产架构（小程序 + 已部署 HIS-Go）

```
┌─────────────────┐     web-view      ┌──────────────────────────────┐
│ 微信小程序        │ ────────────────→ │ https://his.example.com       │
│ his-mp-webview   │                   │   /patient/  → 患者 React H5  │
│ （仅壳，无业务）   │                   │   /api/      → Gateway :8080 │
└─────────────────┘                   └──────────────────────────────┘
```

前置条件：

- 域名 ICP 备案 + **HTTPS**
- 微信公众平台配置业务域名（与 H5 域名一致）
- 患者 H5 内 API 请求使用相对路径 `/api/...`（当前 `client.ts` 已满足）
