# HIS-Go 患者端演示部署指南

## 概述

本文档说明如何部署 HIS-Go 患者端演示环境，包括小程序壳和 H5 患者端的配置。

## 前置条件

- Docker 20.10+
- Docker Compose v2.0+
- Node.js 18+ (仅构建前端时需要)
- 微信开发者工具 (小程序演示)
- Go 1.21+ (仅本地开发时需要)

## 快速开始

### 1. 环境准备

```bash
# 克隆项目
git clone <repository-url>
cd HIS-Go

# 复制环境变量模板
cp docker/.env.demo.example docker/.env.demo

# 根据需要修改环境变量
vim docker/.env.demo
```

### 2. 构建前端 (可选)

```bash
# 构建患者端前端
make build-patient-react

# 或者使用 npm
cd frontend/his-web-patient-react
npm ci --legacy-peer-deps
npm run build
```

### 3. 启动演示服务

#### 方式一：使用 Makefile

```bash
# 启动患者端演示
make demo-patient

# 查看日志
make demo-patient-logs

# 停止服务
make demo-patient-stop
```

#### 方式二：使用脚本

```bash
# 启动服务
./scripts/demo-patient.sh start

# 查看服务状态
./scripts/demo-patient.sh status

# 查看日志
./scripts/demo-patient.sh logs

# 停止服务
./scripts/demo-patient.sh stop
```

#### 方式三：直接使用 Docker Compose

```bash
# 启动服务
docker compose -f docker/docker-compose.demo-patient.yml --env-file docker/.env.demo up -d

# 查看服务状态
docker compose -f docker/docker-compose.demo-patient.yml ps

# 查看日志
docker compose -f docker/docker-compose.demo-patient.yml logs -f

# 停止服务
docker compose -f docker/docker-compose.demo-patient.yml down
```

## 小程序配置

### 1. 导入小程序项目

1. 打开微信开发者工具
2. 选择"导入项目"
3. 项目目录选择 `frontend/his-mp-webview`
4. AppID 使用测试号或真实 AppID

### 2. 配置 H5 地址

编辑 `frontend/his-mp-webview/pages/index/index.js`：

```javascript
// 开发环境
const DEFAULT_PATIENT_H5 = 'http://localhost:5174'

// 云服务器环境
// const DEFAULT_PATIENT_H5 = 'https://your-domain.com/patient/'
```

### 3. 配置项目设置

编辑 `frontend/his-mp-webview/project.config.json`：

```json
{
  "urlCheck": false,
  "es6": true,
  "enhance": true,
  "postcss": true,
  "minified": true
}
```

> **注意**: 开发阶段将 `urlCheck` 设置为 `false` 以允许 HTTP 访问。

### 4. 真机预览

1. 在微信开发者工具中点击"预览"
2. 使用手机微信扫描二维码
3. 确保手机和电脑在同一 WiFi 网络下

## 故障排查

### 服务启动失败

```bash
# 查看服务日志
./scripts/demo-patient.sh logs

# 查看特定服务日志
./scripts/demo-patient.sh logs his-auth

# 检查 Docker 状态
docker ps -a
```

### 小程序无法加载 H5

1. **检查网络连接**
   - 确保手机和电脑在同一 WiFi 网络
   - 检查防火墙设置

2. **检查 H5 地址配置**
   - 确认 `DEFAULT_PATIENT_H5` 配置正确
   - 尝试在浏览器中直接访问 H5 地址

3. **检查 HTTPS 配置**
   - 真机预览需要 HTTPS
   - 开发阶段可使用微信开发者工具

### API 请求失败

```bash
# 检查 Gateway 状态
curl http://localhost:8080/health

# 检查各服务状态
curl http://localhost:8081/health  # Auth
curl http://localhost:8082/health  # User
```

## 云服务器部署

### 1. 服务器准备

```bash
# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 配置防火墙
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

### 2. 部署应用

```bash
# 克隆代码
git clone <repository-url>
cd HIS-Go

# 配置环境变量
cp docker/.env.demo.example docker/.env.demo
vim docker/.env.demo

# 构建前端
make build-patient-react

# 启动服务
make demo-patient
```

### 3. 配置 HTTPS (必须)

小程序真机预览必须使用 HTTPS：

```bash
# 安装 Certbot
sudo apt install certbot

# 获取证书
sudo certbot certonly --standalone -d your-domain.com

# 配置 Nginx SSL (需要修改 nginx.conf)
```

### 4. 配置微信业务域名

1. 登录微信公众平台
2. 进入"开发" -> "开发管理" -> "开发设置"
3. 在"业务域名"中添加你的域名

## 性能优化

### 资源限制

在 `docker-compose.demo-patient.yml` 中添加资源限制：

```yaml
services:
  his-auth:
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 512M
        reservations:
          cpus: '0.25'
          memory: 256M
```

### 数据库优化

```bash
# 调整 PostgreSQL 配置
# 在 docker-compose.yml 中添加 volumes
volumes:
  - ./postgresql.conf:/etc/postgresql/postgresql.conf
```


## 常见问题

### Q: 如何重置演示数据？

```bash
# 停止服务
./scripts/demo-patient.sh stop

# 删除数据卷
docker volume rm his-go_postgres_data

# 重新启动
./scripts/demo-patient.sh start
```

### Q: 如何修改演示账号密码？

1. 修改 `backend/sql/seed_data.sql` 中的密码哈希
2. 重新构建并启动服务

### Q: 小程序真机预览无法加载？

1. 确保使用 HTTPS
2. 确保域名已在微信公众平台配置
3. 确保手机和服务器在同一网络或服务器有公网 IP

### Q: 如何切换 H5 地址？

编辑 `frontend/his-mp-webview/pages/index/index.js`：

```javascript
// 开发环境
const DEFAULT_PATIENT_H5 = 'http://localhost:5174'

// 云服务器环境
const DEFAULT_PATIENT_H5 = 'https://your-domain.com/patient/'
```
