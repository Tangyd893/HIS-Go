# HIS-Go 小程序患者端 — 现场演示脚本

> **形态**：微信小程序壳（`mp-webview`）+ Vue3 H5 患者端（`frontend/patient`）  
> **推荐入口**：微信开发者工具模拟器（最真实）  
> **备用入口**：浏览器 http://localhost/patient  
> **账号**：`demo-patient` / `demo123`（对应患者 **王小明**）

---

## 与管理端的区别

| 维度 | Web 管理端 | 小程序患者端 |
|------|------------|--------------|
| 目录 | `frontend/admin` | `frontend/patient` + `frontend/mp-webview` |
| Compose | `demo-admin.yml` | `demo-patient.yml` |
| 用户 | 医院工作人员 | 患者 |
| 亮点 | 门诊业务闭环 | 自助服务 + **AI 就诊助手** |
| 演示脚本 | `../管理端/` | 本文件 |

两套可独立演示，也可在汇报中先管理端后患者端。

## 演示前准备（10 分钟，首次）

### 1. 启动后端（患者端 Profile）

```bash
docker compose -f deploy/compose/demo-patient.yml \
  --env-file deploy/config/demo.env up -d --build
```

确认容器：`his-gateway-demo`、`his-outpatient-demo`、`his-postgres-demo` 等运行中。

```powershell
curl http://localhost:8080/health   # UP
```

### 2. 构建患者 H5

```bash
cd frontend/patient
npm ci
npm run build
```

产物目录：`frontend/patient/dist`（Nginx 挂载为 `/patient/`）

### 3. 配置微信开发者工具

1. 打开 **微信开发者工具**
2. **导入项目** → 选择 `frontend/mp-webview`
3. AppID：测试号即可
4. **详情 → 本地设置** → 勾选 ✅ **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**
5. 确认 `frontend/mp-webview/pages/index/index.js` 中：
   ```javascript
   const H5_MODE = 'docker'   // 使用 http://127.0.0.1/patient/
   ```
6. 点击 **编译**，模拟器应加载患者端登录页

### 4. 数据自检

```powershell
docker exec his-postgres-demo psql -U his_admin -d his_user -c "SELECT id,name FROM patients WHERE id='patient_001';"
```

应返回王小明。若无数据，执行 seed（同管理端文档）。

### 5. 就诊助手（可选）

`deploy/config/demo.env` 中配置 `DEEPSEEK_API_KEY`、`SILICONFLOW_API_KEY` 可启用完整 RAG；未配置时自动降级为关键词模式，仍可演示。

## 逐步演示

### 0:00 – 1:00　小程序加载

**操作**

- 展示微信开发者工具：左侧项目树 `mp-webview`，右侧模拟器显示患者端

**讲解词**

> 患者端采用「小程序壳 + H5」混合架构。小程序负责微信生态入口与原生能力；业务页面是 Vue3 H5，通过 `web-view` 嵌入。这样一套 H5 可同时服务浏览器和微信小程序，降低双端维护成本。

**技术指向**：`frontend/mp-webview/pages/index/index.wxml` 的 `<web-view>` 组件。

---

### 1:00 – 2:00　患者登录

**操作**

1. 模拟器登录页输入 `demo-patient` / `demo123`
2. 登录后首页显示患者服务中心，用户名为 demo-patient

**讲解词**

> 患者账号与 HIS 用户体系打通，JWT 经网关鉴权。该账号绑定 seed 患者王小明，后续预约、处方数据一致。

---

### 2:00 – 5:00　就诊助手（核心亮点）

**操作**

1. 底部 Tab 点 **助手**（或首页卡片「就诊助手」）
2. 输入：`头痛发热三天，伴有咳嗽`
3. 等待 3–8 秒，阅读 AI 回复
4. 指出：**推荐科室**（如内科）、**就医建议**、**免责声明**

**讲解词**

> 就诊助手是 RAG + 大模型分诊辅助。流程：患者描述症状 → 本地知识库（16 类病症）语义检索 → 结合本院科室表 → DeepSeek 生成建议。强调：**辅助分诊，不能替代医生诊断**。

**技术指向**：
- 前端：`frontend/patient/src/views/triage/TriageChatView.vue`
- 后端：`backend/internal/outpatient/assistant/`
- 知识库：`backend/data/triage/knowledge.json`

| 翻车现象 | 处理 |
|----------|------|
| 「服务暂时不可用」 | 说明降级为关键词模式；检查 `his-outpatient-demo` |
| 等待 >15 秒 | 切下一环节，口头描述能力 |
| 无 API Key | 关键词模式仍可返回科室建议 |

---

### 5:00 – 7:00　预约挂号

**操作**

1. 底部 Tab **挂号**
2. 查看今日排班列表（内科/张医生/上午下午）
3. 若有预约按钮可演示；无则指列表数据来自 `schedule` 服务

**讲解词**

> 患者自助查看号源，减少大厅排队。排班数据与管理端同一后端，号源实时同步。

---

### 7:00 – 9:00　我的处方 + 检查报告

**操作**

1. 底部 Tab **处方** → 查看处方列表
2. 从 **首页** 功能网格进入 **检查报告**（不在底部 Tab）
3. 展示 seed 中的 CT/血常规等演示报告

**讲解词**

> 患者可随时查看医嘱和检查结果，提升就医透明度。

**注意**：若管理端刚演示过开方，处方列表可能有新记录；否则展示 seed 历史处方。

---

### 9:00 – 11:00　健康档案 + 慢病管理

**操作**

1. 底部 Tab **档案** → 就诊时间轴
2. 首页网格 → **慢病管理** → 展示高血压/糖尿病签约（郑十、刘十二）

**讲解词**

> 健康档案聚合历次就诊；慢病管理支持长期随访场景，体现院外患者服务能力。

---

### 11:00 – 13:00　随访计划

**操作**

- 首页网格 → **我的随访** → 查看随访任务列表

**讲解词**

> 随访服务连接院外管理，护士可下发随访计划，患者在端内查看执行。

**注意**：部分随访 API 在网关路由可能未完全注册，若列表为空则口头说明能力已预留。

---

### 13:00 – 15:00　真机预览 + 总结

**操作（可选）**

- 微信开发者工具 → **预览** → 手机扫码（需同 WiFi）
- 或展示 `index.js` 中 `VITE_PATIENT_H5` 切换热更新模式

**讲解词**

> 患者端覆盖预约、结果查询、AI 分诊、慢病随访，与管理端院内流程形成「内外双环」。当前为本地演示版，域名与合规配置上线时需替换。
