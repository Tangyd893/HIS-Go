# HIS-Go Web 管理端 — 8 分钟精简演示脚本

> **适用**：答辩时间紧、只讲管理端、跳过排班生成与扩展菜单  
> **入口**：http://localhost/admin · `demo-admin` / `demo123`

---

## 演示前 2 分钟检查

```powershell
curl http://localhost:8080/health    # UP
```

浏览器登录管理端，**患者管理** 搜「王」确认有 **王小明**。无数据则执行 seed（见 15 分钟版）。

## 8 分钟时间轴

| 时间 | 页面 | 动作 | 一句话讲解 |
|------|------|------|------------|
| 0:00 | 登录页 | `demo-admin` 登录 | Go 微服务 + Vue3 管理端 |
| 0:30 | 挂号管理 | 今天号源 → 挂 **王小明** | 号源扣减、患者搜索 |
| 1:30 | 挂号管理 | **签到** → **开始接诊** | 排队签到一体化 |
| 2:30 | 门诊诊疗 | 主诉+诊断 → 保存 | 诊疗记录落库 |
| 3:30 | 处方管理 | 开处方 → 加 **阿莫西林** → **审核** | 处方审核流转 |
| 5:00 | 收费结算 | **收费** 待支付账单 | 账单状态变更 |
| 6:00 | 药房管理 | **发药** 选处方 | 库存扣减、闭环完成 |
| 7:00 | 处方管理 | 指一下 **已发药** 状态 | 六服务协作总结 |
| 7:30 | — | 结束 | 本地演示版，非生产就绪 |

**总操作点击数**：约 12 次，适合答辩现场。

## 逐步操作（含讲解词）

### 1. 登录（30 秒）

打开 http://localhost/admin → 登录。

> 管理端面向医院工作人员，经 API 网关访问后端微服务。

---

### 2. 挂号 + 签到（2 分钟）

**挂号管理** → 日期今天 → 查询 → 内科上午 → **挂号** → 搜「王」选王小明 → 确认。

下方记录 → **签到** → **开始接诊**。

> 挂号服务管理号源；签到后直接进入接诊，减少窗口重复操作。

**跳过排班**：seed 已有今日号源，不要进排班管理。

---

### 3. 接诊（1 分钟）

**门诊诊疗**（可能已自动跳转）：

- 主诉：`发热咳嗽3天`
- 诊断：`上呼吸道感染`
- 保存

> 诊疗数据写入 clinic 服务，关联后续处方与收费。

---

### 4. 处方 + 审核（1.5 分钟）

**处方管理** → **开具处方** → 王小明 → 添加 **阿莫西林胶囊**（数量2，口服，每日三次，3天）→ 保存 → **审核**。

> 审核是药房发药的前置条件。

---

### 5. 收费（1 分钟）

**收费结算** → 待支付 → **收费**。

> 演示级支付 Stub，展示状态机即可。金额为 0 时口头说明。

---

### 6. 发药 + 收尾（1.5 分钟）

**药房管理** → 阿莫西林 → **发药** → 选刚审核处方 → 确认。

回 **处方管理** 指 **已发药**：

> schedule、registration、clinic、prescription、billing、pharmacy 六个服务完成门诊闭环。系统为 Go 重构版 HIS，Database per Service，Docker Compose 本地可演示。

---

## 不要做的事（8 分钟内）

- ❌ 不要点 EMR、住院、检查检验、随访等
- ❌ 不要演示排班生成（省 1 分钟）
- ❌ 不要展开 Dashboard 服务状态
- ❌ 不要现场改代码或重建 Docker

## 翻车 30 秒救场

| 问题 | 一句话 + 动作 |
|------|----------------|
| 搜不到患者 | 「seed 患者王小明」→ 执行 seed SQL |
| 无号源 | 日期改明天 |
| 无账单 | 「账单随就诊生成」→ 确认处方已审核后刷新 |
| 页面空白 | 刷新；检查对应 `his-*-demo` 容器 |
