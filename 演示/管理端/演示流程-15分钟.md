# HIS-Go Web 管理端 — 15 分钟现场演示脚本

> **入口**：http://localhost/admin  
> **账号**：`demo-admin` / `demo123`  
> **固定患者**：王小明（`patient_001`）  
> **固定药品**：阿莫西林胶囊  
> **固定科室**：`dept_001`（内科）

---

## 演示前 5 分钟（幕后准备）

```powershell
docker version                          # 必须有 Server 段
curl http://localhost:8080/health       # {"status":"UP"}
curl -o NUL -w "%{http_code}" http://localhost/admin/    # 200
```

**登录自检**：`demo-admin` / `demo123`  
**数据自检**：**患者管理** → 搜索「王」→ 应出现 **王小明**

若列表为空：

```powershell
docker exec his-postgres-demo psql -U his_admin -f /sql-init/seed_data.sql
docker exec his-postgres-demo psql -U his_admin -f /sql-init/seed_data_extended.sql
```

**Docker 引擎 500 错误时**：`docker desktop restart`，等待 30 秒后再试。

# 第一幕：业务闭环（约 10 分钟）

### 0:00 – 0:45　开场 + 登录

**操作**

1. 打开 http://localhost/admin
2. 登录 `demo-admin` / `demo123`

**讲解词**

> 这是 HIS-Go 医院信息管理系统的 Web 管理端。后端采用 Go 微服务，API 网关统一入口；前端 Vue 3 + Ant Design。今天演示门诊主路径：**排班 → 挂号 → 接诊 → 处方 → 收费 → 发药**。

---

### 0:45 – 1:30　工作台（快速带过）

**操作**：停留在 **工作台**，指一下统计卡片和快捷入口。

**讲解词**

> 工作台汇总业务概览。实际演示走左侧菜单，按医院真实工作流操作。

**注意**：Dashboard 部分服务状态为演示数据，勿展开，直接进入业务菜单。

---

### 1:30 – 2:30　排班管理

**方案 A（推荐，省时间）**

seed 已含今日内科排班，口头说明：

> 张医生今天内科上午/下午号源已就绪，直接进入挂号。

**方案 B（展示生成能力）**

1. **排班管理** → **生成排班**
2. 开始/结束日期选 **今天**
3. 科室 ID 填 `dept_001`
4. 确认 → 表格显示号源（总号源 / 已约）

**讲解词**

> 排班服务按科室、时段发布号源，为挂号提供库存。

---

### 2:30 – 4:30　挂号管理

**操作**

1. **挂号管理**
2. 号源区：日期选 **今天** → **查询**
3. 选 **内科 · 上午** → **挂号**
4. 患者搜索 **「王」** → 选 **王小明** → 确认
5. 挂号记录：**待签到** → **签到**
6. **已签到** → **开始接诊**

**讲解词**

> 挂号连接排班与用户服务。签到后一键跳转接诊，减少重复录入。

| 翻车现象 | 处理 |
|----------|------|
| 搜不到患者 | 搜「王」或 `13900000001`；仍无则执行 seed 命令 |
| 今天无号源 | 改明天或走排班方案 B |
| 按钮无反应 | 刷新页面；检查 `his-registration-demo` 容器 |

---

### 4:30 – 6:00　门诊诊疗

**操作**

- 「开始接诊」后应跳转 **门诊诊疗** 并带入患者
- 若未跳转：**门诊诊疗** → **新建诊疗记录** → 搜 **王小明**
- 主诉：`发热、咳嗽 3 天`
- 诊断：`上呼吸道感染`
- 保存

**讲解词**

> 诊疗记录是电子病历基础单元，后续处方、收费关联本次就诊。

---

### 6:00 – 8:00　处方管理

**操作**

1. **处方管理** → **开具处方**
2. 患者：**王小明**
3. **添加药品** → 搜 **阿莫西林** → 选「阿莫西林胶囊」
4. 数量 `2`，用量 `0.5g`，用法 `口服`，频次 `每日三次`，天数 `3`
5. 保存 → **待审核** → 点 **审核** → **已审核**

**讲解词**

> 处方审核通过后才允许药房发药，体现监管要求。

---

### 8:00 – 9:00　收费结算

**操作**

1. **收费结算**
2. 找王小明、**待支付** 账单 → **收费** → **已支付**

**讲解词**

> 演示环境支付为 Stub，重点展示账单状态流转。

| 翻车现象 | 处理 |
|----------|------|
| 列表为空 | 确认处方已审核，刷新；口头说明账单在就诊后生成 |
| 金额为 0 | 说明「演示级计费，后续接价表」 |

---

### 9:00 – 10:00　药房管理

**操作**

1. **药房管理**
2. **阿莫西林胶囊** → **发药**
3. 下拉选刚审核的处方 → 数量 `2` → 确认
4. 回 **处方管理** 确认状态 **已发药**

**讲解词**

> 发药校验处方并扣库存，挂号到取药全流程可追溯。六个微服务经网关协作完成闭环。
